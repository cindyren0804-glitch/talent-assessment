## 一、产品定位

**一句话**：AI驱动的动态天赋测评系统，用"迫选+时压"机制测出真实天赋排序。

**核心差异点**：
- ❌ 固定题库 → ✅ 动态生成
- ❌ 单一维度 → ✅ 画像匹配
- ❌ 自我报告量表 → ✅ 迫选碰撞机制

## 二、底层逻辑

### 2.1 迫选机制（Forced-Choice）

**原理**：把两个**同等正向**或**同等负向**的描述放在一起，强制放弃一个。

**为什么有效？**
```
传统量表："你喜欢学习吗？1-5分"
问题：社会期望偏差 → 大家都选4-5分

迫选设计：
A. 我喜欢读深奥的书，理解复杂理论
B. 我喜欢认识厉害的人，拓展人脉
效果：必须放弃一个 → 逼出真实偏好
```

**题目设计原则**：
| 原则 | 说明 | 示例 |
|------|------|------|
| 等价性 | 两个选项社会期望值相等 | 都是正向描述 |
| 对立性 | 代表不同的天赋维度 | 战略思维 vs 影响力 |
| 场景化 | 具体行为而非抽象概念 | "开会时我会..." vs "我喜欢..." |

### 2.2 时间压力（Time Pressure）

**20秒倒计时的心理学原理**：
```
系统1（快思考）────┐
    直觉反应      │
    情感驱动      ├──→ 时间压力下激活
    经验模式      │
                  │
系统2（慢思考）────┘
    理性计算      ←──→ 时间压力下被抑制
    逻辑分析
    自我包装
```

**产品设计**：
- 每题限时 20 秒
- 可视化倒计时进度条
- 超时自动跳过（记为"无偏好"）

### 2.3 主题矩阵碰撞

**盖洛普模型**：34个主题两两配对 → C(34,2) = 561 种组合 → 实际使用 177 题

**我们的简化模型**：提炼核心维度

```
              输入端                    处理端                    输出端                    动力端
           ┌─────────┐              ┌─────────┐              ┌─────────┐              ┌─────────┐
   广度    │  搜集   │              │  分析   │              │  系统   │              │  成就   │    外在
   涉猎    │         │              │         │              │  构建   │              │         │    驱动
           └────┬────┘              └────┬────┘              └────┬────┘              └────┬────┘
                │                        │                        │                        │
           ═════╪════════════════════════╪════════════════════════╪════════════════════════╪═════
                │                        │                        │                        │
           ┌────┴────┐              ┌────┴────┐              ┌────┴────┐              ┌────┴────┐
   深度    │  专注   │              │  共情   │              │  执行   │              │  意义   │    内在
   死磕    │         │              │         │              │         │              │         │    驱动
           └─────────┘              └─────────┘              └─────────┘              └─────────┘
```

**碰撞矩阵**：
- 4 个维度 × 每维度 2 个极点 = 8 个核心节点
- 两两配对：C(8,2) = 28 种基础组合
- 加上三向碰撞（进阶版）：C(8,3) = 56 种

### 4.3 动态生成算法

**核心算法**：
```python
def generate_questionnaire(user_profile, num_questions=20):
    # 1. 计算主题权重
    theme_weights = calculate_theme_weights(user_profile)

    # 2. 选择碰撞组合
    pairs = select_collision_pairs(theme_weights, num_questions)

    # 3. 生成题目
    questions = []
    for theme_a, theme_b in pairs:
        # 获取场景
        scenario = match_scenario(user_profile)

        # 生成题目
        q = generate_forced_choice(
            theme_a,
            theme_b,
            scenario,
            user_profile['language_style']
        )
        questions.append(q)

    # 4. 难度排序
    questions = sort_by_difficulty(questions)

    return questions
```

### 4.4 题目模板库

**输入端对立**：
| 主题A | 主题B | 基础模板 |
|-------|-------|----------|
| 广度涉猎（搜集） | 深度死磕（专注） | A：我喜欢接触新领域、新事物 / B：我喜欢在一个领域深耕到底 |
| 信息吸收 | 信息过滤 | A：我倾向于尽可能多地收集信息 / B：我倾向于只看最关键的信息 |

**处理端对立**：
| 主题A | 主题B | 基础模板 |
|-------|-------|----------|
| 逻辑推演（分析） | 情绪感知（共情） | A：做决定时我更依赖数据和逻辑 / B：做决定时我更依赖直觉和感受 |
| 系统思维 | 敏捷应变 | A：我喜欢制定详细计划再行动 / B：我习惯边做边调整 |

**输出端对立**：
| 主题A | 主题B | 基础模板 |
|-------|-------|----------|
| 建立规则（系统） | 具体执行（执行） | A：我更擅长设计流程和规则 / B：我更擅长亲自把事情落地 |
| 影响他人 | 独立贡献 | A：我享受说服和影响别人的过程 / B：我更喜欢专注做好自己的事 |

**动力端对立**：
| 主题A | 主题B | 基础模板 |
|-------|-------|----------|
| 成就驱动 | 意义驱动 | A：我追求把事情做成、拿到结果 / B：我追求做的事情有意义、有价值 |
| 外在认可 | 内在满足 | A：被认可和赞赏会让我更有动力 / B：我更在意自己内心的满足感 |

### 5.3 计分逻辑

**碰撞计分法**：
```python
def calculate_scores(answers):
    # 每个主题初始化
    themes = {t: 0 for t in ALL_THEMES}

    for answer in answers:
        theme_a, theme_b = answer['pair']
        choice = answer['choice']  # 'a', 'b', or 'neutral'
        strength = answer['strength']  # 1-5 强度

        if choice == 'a':
            themes[theme_a] += strength
            themes[theme_b] -= (strength * 0.5)  # 输方扣分
        elif choice == 'b':
            themes[theme_b] += strength
            themes[theme_a] -= (strength * 0.5)
        # 中立则不计分

    # 排序
    ranked = sorted(themes.items(), key=lambda x: x[1], reverse=True)
    return ranked
```

## 十一、5Why 深挖机制（柔和版）

### 11.1 机制概述

**目的**：从表面选择追溯到深层信念/早期经历，用温柔的方式帮助用户理解自己

**触发条件**：
1. 矛盾纠结点：同一维度内多次 A/B 摇摆，或跨维度出现相反高分
2. 动力端深挖：成就驱动 vs 意义驱动选择比例接近（40%-60%）
3. 用户主动选择："想深入了解自己"

### 11.2 两个深挖方向

#### 方向 B：矛盾纠结点深挖
**常见矛盾组合**：
| 矛盾组合 | 表面现象 | 探索方向 |
|---------|---------|---------|
| 广度涉猎 ↔ 深度死磕 | 收藏很多但不深入 | 探索"不想错过"背后的期待 |
| 系统构建 ↔ 执行力 | 想规划但迟迟不动 | 探索"确定感"对用户的意义 |
| 成就驱动 ↔ 意义驱动 | 为认可做事但内心空虚 | 探索动力来源的温柔故事 |
| 逻辑推演 ↔ 情绪感知 | 讲道理但伤人/太共情 | 探索保护自己的方式 |

#### 方向 C：动力端深挖
**基于题目**：Q4、Q7、Q8、Q16-20 共 7 道动力相关题
**探索目标**：理解"为什么被认可/有意义感对用户重要"

### 11.3 提问设计原则（柔和版）

| 层级 | 目标 | 提问特点 |
|-----|------|---------|
| Why 1 | 建立连接 | 用"我注意到""看起来"观察式开场 |
| Why 2 | 探索感受 | 用"What"代替"Why"减少防御 |
| Why 3 | 理解重视 | 好奇"这对你重要在哪里" |
| Why 4 | 发现模式 | 邀请回顾，不强行连接 |
| Why 5 | 温柔总结 | 用"原来""也许"等柔和词 |

**柔和化技巧**：
- ✅ "我注意到你..." → 观察而非评判
- ✅ "能多说一点吗" → 邀请而非追问
- ✅ "对你来说意味着" → 理解而非分析
- ✅ "原来...呢" → 发现而非定论
- ❌ 避免负面词汇：焦虑、害怕、痛苦 → 改用：担心、不确定、想更好

### 12.4 第三部分：内在模式（5Why 深挖）

**定位**：探索"为什么"——通过5Why深挖发现行为背后的深层动机

**触发条件**：
1. 矛盾纠结点：同一维度内多次 A/B 摇摆
2. 动力端深挖：成就驱动 vs 意义驱动选择比例接近（40%-60%）
3. 用户主动选择："想深入了解自己"

**与第二部分的区别**：
| 维度 | 第二部分（天赋详解） | 第三部分（内在模式） |
|------|---------------------|---------------------|
| 问题 | 是什么 | 为什么 |
| 性质 | 客观描述 | 主观探索 |
| 示例 | 系统构建者擅长看见结构 | 你追求系统是因为需要确定感 |
| 来源 | 天赋定义 | 5Why对话挖掘 |

```markdown
## 🔮 你的内在模式

### 我们发现的
在你的选择中，我注意到一个挺有意思的模式：

**你既喜欢广泛了解很多东西，又想把一件事做到极致。**

这种多元的好奇心和深耕的渴望在你身上和谐共存，只是有时候会让你有点纠结～

### 温柔的探索
带着好奇，我们聊了一会儿，发现了一些有趣的事情：

**原来你的好奇心背后，还藏着一份想要被看见、被认可的小期待呢。**

**发现路径**：
不想错过 → 希望跟大家同步 → 被看见的感觉很好 → 从小被关注会带来开心

### 这背后的影响
这个模式可能会让你：
- 在信息收集时停不下来，因为"万一有用呢"
- 开始做一件事时充满热情，但深入时担心错过其他
- 完成一个项目后，需要外部的反馈才能感到满足

### 想对你说
这种想要被看见的心情，其实是人之常情呢。
它让你保持好奇心，不断学习新东西。
同时，也别忘了：你的价值不只是"知道很多"，
"在一件事上有洞见"同样闪闪发光 ✨
```

### 12.5 第四部分：礼物与挑战（天赋组合效应）

**定位**：展示天赋组合的化学反应——1+1>2 的协同效应，或潜在冲突

**与第二、三部分的区别**：
| 维度 | 第二部分 | 第三部分 | 第四部分（本部分） |
|------|---------|---------|------------------|
| 视角 | 单个天赋 | 深层动机 | 天赋组合 |
| 问题 | 是什么 | 为什么 | 组合后产生什么 |
| 示例 | 系统构建者擅长框架 | 追求系统是因为需要确定感 | 系统+分析=问题拆解专家 |
| 性质 | 独立描述 | 主观探索 | 协同/冲突分析 |

```markdown
## 🎁 你的礼物与挑战

### 天赋组合带给你的礼物

**系统构建 + 分析**
这两者的组合让你成为团队里的"问题拆解专家"。
别人看到一个烂摊子，你看到的已经是分类好的模块和解决路径。

**专注 + 成就驱动**
这个组合让你是"结果导向的执行者"。
不仅能埋头做事，还能确保事情有交代、有成果。

**广度涉猎的存在**
虽然它排第五，但给你的系统思维提供了丰富的素材库。
你能把 A 领域的方法论迁移到 B 领域，这是你的隐藏优势。

---

### 可能遇到的小挑战

**当系统构建遇上广度涉猎**
你容易陷入"资料收集 → 整理框架 → 发现新领域 → 再收集"的循环，
导致真正动手的时间被压缩。

**当成就驱动遇上专注**
你会很在意"这件事做成能不能被看到"，
如果感觉没人关注，可能连开始的动力都没有。

**关系建立维度偏弱**
在追求效率和结果时，可能会忽略同事的情绪和关系维护，
让人觉得你有点"冷"。
```

### 12.6 第五部分：温柔建议

**定位**： actionable 的行动指导——基于前四部分分析给出具体建议

**与前四部分的关系**：
- 第一部分（角色概览）→ 让你知道"我是谁"
- 第二部分（天赋详解）→ 让你了解"我有什么"
- 第三部分（内在模式）→ 让你理解"我为什么这样"
- 第四部分（礼物与挑战）→ 让你看清"组合效应"
- **第五部分（温柔建议）→ 告诉你"该怎么办"**

**建议设计原则**：
1. **基于证据**：每个建议都对应前文的分析
2. **具体可行**：不是"要自信"而是"在会议前写下3个你想分享的观点"
3. **温柔语气**：用"试试""也许""可以考虑"代替"必须""应该"

```markdown
## 💝 给你的温柔建议

### 发挥你的系统构建天赋
1. **主动承担流程设计**：团队需要新流程时，你就是最佳人选
2. **可视化你的思考**：多画思维导图、流程图，让大家跟上你的思路
3. **允许"足够好"**：80分的系统比100分的计划更有价值

### 平衡广度与深度
1. **战略选点**：选定1-2个真正感兴趣的方向深耕，其他的保持轻松涉猎
2. **试试单点突破**：给自己一个小实验，比如"这个季度只专注一个技能"
3. **重新定义"被看见"**：你的价值不只是"知道很多"，专业深度同样吸引人

### 滋养你的动力
1. **建立内在记分牌**：问问自己"如果没人知道，我还会想做吗？"
2. **给自己鼓掌**：设置一些属于自己的小里程碑
3. **试试纯粹的事**：偶尔做一些只是为了开心、不计结果的事

### 成长方向
🌱 练习接受"不确定"和"没条理"，不是所有事情都需要系统
🌱 探索"不被看见时的动力"，找到内在驱动的火花
🌱 允许自己"只想不做"一段时间，休息也是成长的一部分
```
