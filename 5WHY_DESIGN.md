# 天赋测评 - 5Why 深挖机制设计

> 版本：v1.1 | 日期：2026-03-07 | 说明：柔和版提问设计

---

## 一、机制概述

**目的**：从表面选择追溯到深层信念/早期经历/核心恐惧

**核心逻辑**：
```
表面选择 → 第一次 Why → 动机层 → 第二次 Why → 价值观层 → 
第三次 Why → 信念层 → 第四次 Why → 经历层 → 第五次 Why → 核心叙事
```

## 二、方向 B：矛盾纠结点深挖

### 2.1 识别矛盾

**触发条件**：
- 同一维度内多次出现 A/B 摇摆
- 用户反馈"两个都像""看情况"
- 跨维度出现相反倾向（如既想广度涉猎又想深度死磕）

**常见矛盾组合**：

| 矛盾组合 | 表面现象 | 深层可能 |
|---------|---------|---------|
| 广度涉猎 ↔ 深度死磕 | 收藏很多但不深入 / 想深入但怕错过 | FOMO 恐惧 / 完美主义 |
| 系统构建 ↔ 执行力 | 想规划完美但迟迟不开始 / 做了但觉得乱 | 控制欲 vs 行动焦虑 |
| 成就驱动 ↔ 意义驱动 | 为认可做事但内心空虚 / 追求意义但没成果 | 外部验证依赖 / 理想主义 |
| 逻辑推演 ↔ 情绪感知 | 讲道理但伤人 / 太共情但没边界 | 理性防御 / 情感淹没 |

### 2.2 5Why 提问链（示例：广度涉猎 vs 深度死磕矛盾）

```
【起点】我注意到你既喜欢广泛了解很多东西，也喜欢深入研究一个领域，这两种状态在你身上好像都有～
    ↓
Why 1: 嗯，确实两个都挺吸引我的，不过有时候也会纠结
    ↓
What 1: 这种纠结的时候，你通常是什么感受呢？
    ↓
Why 2: 我会担心如果选了一个，另一个就顾不上了
    ↓
What 2: 如果只能顾一个，那种"顾不上"的感觉会让你想到什么？
    ↓
Why 3: 好像会有一种被落下的感觉，大家聊的我不知道
    ↓
What 3: 那种"跟大家不同步"的感觉，对你来说意味着什么呢？
    ↓
Why 4: 可能觉得不容易融入，或者显得自己不够厉害
    ↓
What 4: 这种希望自己"能跟上"、"够厉害"的想法，你觉得是从什么时候开始有的呢？
    ↓
Why 5: 回想起来，可能是上学的时候吧，知道得多会被老师同学注意到
    ↓
【核心洞察】原来你的好奇心背后，还有一个想要被看见、被认可的小期待呢
```

### 2.3 提问设计原则（柔和版）

| 层级 | 目标 | 提问特点 | 柔和技巧 |
|-----|------|---------|---------|
| Why 1 | 建立连接 | 观察式开场，不带评判 | 用"我注意到""看起来"代替质问 |
| Why 2 | 探索感受 | 聚焦"是什么感受/想到什么" | 用"What"代替"Why"减少防御 |
| Why 3 | 理解重视 | 好奇"这对你重要在哪里" | 避免负面词汇（焦虑、害怕、痛苦） |
| Why 4 | 发现模式 | 邀请回顾，不强行连接 | 用"你觉得是什么时候"代替"因为你小时候" |
| Why 5 | 温柔总结 | 用"原来""也许"等柔和词 | 给用户确认空间，不武断下结论 |

**柔和化技巧**：
- ✅ "我注意到你..." → 观察而非评判
- ✅ "能多说一点吗" → 邀请而非追问
- ✅ "对你来说意味着" → 理解而非分析
- ✅ "原来...呢" → 发现而非定论
- ❌ 避免："为什么你这么焦虑/害怕/难受"

## 三、方向 C：动力端深挖

### 3.1 动力端测评题目回顾

**Q4: 成就驱动 vs 意义驱动**
- A. 虽然不感兴趣，但做好了能获得认可，我就能投入去做
- B. 只有觉得这件事有意义/有价值，我才有动力做好

**相关交叉题**：
- Q7: 系统构建 vs 成就驱动
- Q8: 执行力 vs 意义驱动
- Q16: 情绪感知 vs 成就驱动
- Q17: 情绪感知 vs 意义驱动
- Q18: 系统构建 vs 意义驱动
- Q19: 执行力 vs 成就驱动
- Q20: 广度涉猎 vs 成就驱动

### 3.2 5Why 提问链（示例：成就驱动型）

```
【起点】看起来你挺在意把事情做成、拿到好结果的，这种动力对你来说意味着什么呢？
    ↓
Why 1: 做好了会感觉踏实，有成就感
    ↓
What 1: 那种踏实的感觉，具体是什么样的呢？
    ↓
Why 2: 就是觉得自己的努力有了着落，没有白费
    ↓
What 2: "没有白费"对你来说是挺重要的一件事吧？能多说一点吗？
    ↓
Why 3: 嗯，如果不确定有没有用，就很难安心去做
    ↓
What 3: 那种"不确定"的状态，会让你有什么感觉？
    ↓
Why 4: 会有点慌，好像漂浮着没有抓手
    ↓
What 4: 这种想要"有抓手"、"有确定感"的模式，你觉得是什么时候开始的呢？
    ↓
Why 5: 可能是小时候吧，做了事情如果能被看到、被夸，就会很开心
    ↓
【核心洞察】原来你追求成果的背后，有一份想要被看见、被肯定的温柔期待呢
```

### 3.3 5Why 提问链（示例：意义驱动型）

```
【起点】你选择工作的时候挺看重意义感的，能说说对你来说"有意义"是什么样的感觉吗？
    ↓
Why 1: 就是觉得做的事情有价值，不是在重复无意义的动作
    ↓
What 1: 那种"有价值"的感觉，对你来说重要在哪里呢？
    ↓
Why 2: 会让我觉得时间花得值得，心里比较充实
    ↓
What 2: "心里充实"这种状态，你通常在什么情况下会感受到？
    ↓
Why 3: 当我觉得这件事不只是为了完成，而是真的有影响的时候
    ↓
What 3: 那种"真的有影响"的感觉，对你来说特别珍贵吧？
    ↓
Why 4: 嗯，会感觉自己的存在是有分量的，不是可有可无
    ↓
What 4: 这种希望自己"有分量"的期待，你觉得是从什么时候开始觉察到的？
    ↓
Why 5: 可能是慢慢意识到的吧，不想只是随波逐流地过
    ↓
【核心洞察】原来你追求意义的背后，是一份想要活得有分量、有独特价值的渴望呢
```

## 四、5Why 实施流程

### 4.1 自动触发条件

```python
# 伪代码
def should_trigger_5why(answers):
    # 条件1: 动力端 A/B 选择比例接近（40%-60%之间）
    if achievement_vs_meaning_ratio in [0.4, 0.6]:
        return True, "动力端矛盾"
    
    # 条件2: 同一维度内出现3次以上摇摆
    if dimension_inconsistency_count >= 3:
        return True, "维度内矛盾"
    
    # 条件3: 跨维度出现相反高分
    if broad_exploration_high and deep_focus_high:
        return True, "广度深度矛盾"
    
    # 条件4: 用户主动选择"深入探索"
    if user_choice == "深挖模式":
        return True, "用户主动"
    
    return False
```

### 4.2 对话流程设计

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 呈现初步结果                                        │
│  "根据你的选择，你倾向于【成就驱动型】，但在意义感题目上        │
│   也有犹豫。想不想深入看看这背后的原因？"                     │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 用户确认                                            │
│  [开始深挖] [先看角色报告] [跳过]                             │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 5Why 对话（5轮问答）                                │
│  每轮一个 Why，根据用户回答动态调整下一个问题                  │
│  使用自然语言，避免机械感                                     │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 总结洞察                                            │
│  呈现核心叙事 + 可能的盲区 + 发展建议                         │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 对话设计要点（柔和版）

| 要点 | 说明 | 示例 |
|-----|------|------|
| 温暖开场 | 像朋友关心一样开启话题 | "我注意到你选了 A，同时 B 好像也有点像你对吗～" |
| 邀请而非追问 | 给用户选择权，不强迫 | "想不想多聊聊这个？不想也没关系" |
| 即时共鸣 | 每轮让用户感到被理解 | "听起来那个感觉挺微妙的～" "原来是这样呀" |
| 正面词汇 | 避免负面表述 | ❌ "为什么你这么焦虑" → ✅ "那种不确定的感觉是怎样的" |
| 模式发现 | 第4-5轮 gentle 地探索 | "这种模式，你觉得是从什么时候开始的呢？" |
| 赋权确认 | 让用户拥有最终解释权 | "这是我感受到的，你觉得呢？" "有没有哪里不太对？" |
| 随时退出 | 尊重用户的节奏 | "这个话题要是有点深，我们可以先放一放～" |

## 五、输出格式

```markdown
## 🔀 你的内在矛盾：广度涉猎 vs 深度死磕

### 表面现象
你既喜欢广泛了解多个领域，又想把一件事做到极致，这种多元的好奇心和深耕的渴望在你身上和谐共存，只是有时候会让你有点纠结～

### 5Why 温柔发现
**核心洞察**：你的好奇心背后，还藏着一份想要被看见、被认可的小期待

**发现路径**：
不想错过 → 希望跟大家同步 → 被看见的感觉很好 → 从小被关注会带来开心

### 这带给你的
✨ **礼物**：知识面宽，能快速建立跨领域连接，是团队里的"知识枢纽"
🌱 **小挑战**：有时候会在选择上犹豫，担心顾此失彼

### 温柔建议
1. **战略选点**：选定1-2个真正感兴趣的方向深耕，其他的保持轻松涉猎
2. **重新定义"被看见"**：你的价值不只是"知道很多"，"在一件事上有洞见"同样闪闪发光
3. **试试单点突破**：给自己一个小实验，比如"这个季度只专注一个技能"，看看会发生什么
```

### 5Why 温柔发现
**核心洞察**：你追求成果的背后，有一份想要被看见、被肯定的温柔期待

## 六、技术实现要点

### 6.1 AI Prompt 结构（柔和版）

```
你是一位温暖的天赋测评陪伴者，正在和用户进行一场轻松的探索对话。

【用户背景】
- 测评结果：{top_themes}
- 矛盾点/动力端选择：{focus_area}

【当前轮次】：第 {n} 轮
【上一条用户回答】：{last_answer}

【对话风格】
- 像朋友聊天一样自然，不要用专业术语
- 多用"呢""呀""吧""～"等语气词
- 用观察代替质问（"我注意到""看起来"）
- 用 What 代替 Why（"这意味着什么"代替"为什么"）
- 避免负面词汇：焦虑、害怕、痛苦、无用、失败
- 用积极词汇代替：担心、不确定、想更好、期待

【任务】
1. 基于用户回答，问出下一个温和的探索问题
2. 给用户"被理解"的感觉，可以说"听起来..."
3. 如果用户不想深入，尊重说"没关系，我们换个话题"
4. 第4-5轮可以 gently 问："这种模式，你觉得是从什么时候开始的呢？"

【输出】
只输出问句，像朋友发微信一样自然，不要带标签和解释。

【示例】
用户：我挺在意结果的
AI：听起来你做事希望有个着落～那种"稳了"的感觉对你意味着什么呢？
```

### 6.2 数据结构

```typescript
interface FiveWhySession {
  sessionId: string;
  userId: string;
  triggerType: 'contradiction' | 'motivation' | 'user_choice';
  focusArea: string; // 例如："广度深度矛盾" "成就驱动"
  
  rounds: {
    roundNumber: number;
    question: string;
    answer: string;
    insight: string; // AI 识别的模式
  }[];
  
  finalInsight: {

    rootBelief: string;

  };

}
```

## 七、边界与伦理

### 7.1 安全边界

| 情况 | 处理方式 |
|-----|---------|
| 用户不想继续 | 随时可退出，保留已有结果 |
| 触及创伤经历 | 温和转移："这个可能有点深，我们先放一放" |
| 用户情绪激动 | 共情确认 + 建议专业帮助 |
| AI 不确定 | 诚实表达："我不太确定，你觉得呢？" |

### 7.2 免责声明

```
5Why 深挖是为了帮助你更好地理解自己，结果仅供参考。
如果触及到让你不适的内容，请随时停止，必要时寻求专业心理咨询。
```
